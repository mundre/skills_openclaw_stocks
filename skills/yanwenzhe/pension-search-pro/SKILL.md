# 企业年金智能查询技能 Pro (pension-search-pro)

**版本**：v1.0.0  
**作者**：OpenClaw User  
**描述**：企业年金/职业年金智能查询技能。自动识别单位性质，精确判断年金类型，18 种调查渠道 + 多搜索引擎深度搜索，输出带来源链接和错误检查的标准化调查报告。

---

## 功能特性

### 🔍 核心能力
- **单位性质智能识别**：自动判断事业单位/企业/民营企业
- **年金类型精确判断**：企业年金 vs 职业年金
- **18 种调查渠道**：政府采购网、银行官网、招聘信息、员工分享等
- **多搜索引擎深度搜索**：Tavily + 百度/Bing/360/搜狗 4 引擎
- **标准化报告输出**：带来源链接、置信度、错误检查清单

### 📋 调查渠道（18 种）

| 优先级 | 渠道类型 | 具体来源 |
|--------|----------|----------|
| ⭐⭐⭐⭐⭐ | 政府采购网 | 深圳政府采购网、阳光采购平台 |
| ⭐⭐⭐⭐⭐ | 银行官网 PDF | 工商银行、建行、招行等托管银行 |
| ⭐⭐⭐⭐ | 部门决算/预算 | 财政局官网、单位公开预算 |
| ⭐⭐⭐⭐ | 单位官网 | 官网历史沿革、单位性质说明 |
| ⭐⭐⭐ | 招聘信息 | 高校就业网、智联、BOSS、猎聘 |
| ⭐⭐ | 员工分享 | 知乎、脉脉、看准网（需交叉验证） |

## 使用方法

### 基本搜索
```bash
./search.sh "企业名称"
```

### 指定搜索结果数量
```bash
./search.sh "企业名称" 10
```

### 完整调查（所有渠道）
```bash
./search.sh "企业名称" --full
```

### 仅查询单位性质
```bash
./search.sh "企业名称" --nature-only
```

## 工具使用顺序（强制执行）

### 🔍 第一梯队：Tavily API（必须先用）

**8 个关键词组合全部搜索**：
| 序号 | 关键词 | 目的 |
|------|--------|------|
| 1 | 企业名 + 企业年金/职业年金 | 直接确认年金 |
| 2 | 企业名 + 托管人/受托人 | 找开户银行 |
| 3 | 企业名 + 招标/采购 年金 | 找招标公告 |
| 4 | 企业名 + 预算/决算 年金 | 找财政公开 |
| 5 | 企业名 + 单位性质 事业单位 | 确认单位类型 |
| 6 | 企业名 + 招聘 待遇 五险一金 | 找招聘信息 |
| 7 | 企业名 + 福利 企业年金 | 找员工福利 |
| 8 | 上级单位 + 下属单位 + 预算 | 找上级预算 |

### 🔍 第二梯队：Multi Search Engine（必须全部执行）

**4 个搜索引擎都必须用，不能跳过任何一个**：
| 序号 | 搜索引擎 | URL 模板 | 必须执行 |
|------|----------|----------|----------|
| 1 | **百度** | `https://www.baidu.com/s?wd={keyword}` | ✅ |
| 2 | **Bing** | `https://cn.bing.com/search?q={keyword}&ensearch=0` | ✅ |
| 3 | **360** | `https://www.so.com/s?q={keyword}` | ✅ |
| 4 | **搜狗** | `https://sogou.com/web?query={keyword}` | ✅ |

## 企业名称变更说明（强制要求）

**如果输入的企业名称 ≠ 实际搜索的名称，必须说明原因**：

| 情况 | 说明模板 | 示例 |
|------|----------|------|
| **企业改名** | "该企业于 XXXX 年从「原名称」更名为「现名称」" | 深圳市道桥维修中心 → 深圳市路桥建设集团有限公司 |
| **使用子公司名称** | "该企业为「母公司」的子公司/下属单位" | 深圳市道桥维修中心 → 深圳市路桥集团下属单位 |
| **使用集团名称** | "该企业隶属于「集团名称」，搜索时使用集团名称" | 深圳市创意文化中心 → 深圳市文化集团 |
| **使用简称** | "「全称」的常用简称为「简称」" | 腾讯科技（深圳）有限公司 → 腾讯 |
| **事业单位改制** | "该单位于 XXXX 年从事业单位改制为企业" | 深圳市 XX 中心 → 深圳市 XX 有限公司 |

## 单位性质判断规则

| 单位类型 | 年金类型 | 缴费比例 | 强制/自愿 |
|----------|----------|----------|----------|
| 事业单位（编制内） | 职业年金 | 单位 8% + 个人 4% | 强制 |
| 企业 | 企业年金 | 单位≤8% + 个人≤4% | 自愿 |
| 民营企业 | 企业年金（自愿） | 单位≤8% + 个人≤4% | 自愿 |
| 国企/央企 | 企业年金 | 单位≤8% + 个人≤4% | 自愿 |
| 事业单位企业化管理 | 需具体确认 | 可能两者皆有 | 需确认 |

**关键词优先级**：
1. 公告中明确提及"企业年金"或"职业年金" → 直接采用
2. 无明确提及 → 根据单位性质推断
3. 特殊情况（如企业化管理事业单位）→ 单独标注说明

## 置信度标注规则

| 星级 | 说明 | 证据要求 |
|------|------|----------|
| ⭐⭐⭐⭐⭐ | 确凿 | 官方公告 + 银行 PDF 双重确认 |

| ⭐⭐ | 低置信 | 单一来源或员工分享 |
| ⭐ | 极低 | 传闻或无直接证据 |

## 依赖要求

### 必需
- `curl` - HTTP 请求
- `jq` - JSON 处理
- `bash` 5.0+

### 可选
- `pypdf` - PDF 解析（Python 库）
- `pdftotext` - PDF 转文本工具
- Tavily API Key - 深度搜索（推荐）

### 安装依赖
```bash
# Ubuntu/Debian
sudo apt-get install curl jq

# macOS
brew install curl jq

# Python PDF 解析
pip install pypdf
```

## 禁止行为

| ❌ 禁止行为 | ✅ 正确做法 |
|------------|------------|
| 跳过 Tavily 直接用其他工具 | Tavily 必须第一 |
| Multi Search Engine 只用部分引擎 | 百度/Bing/360/搜狗全部执行 |
| 只 fetch 搜索结果页 | 逐个访问前 20 条链接 |
| 拿到 PDF 说"无法解析" | 用 pypdf/strings/pdftotext 自己解析 |
| 单一来源下结论 | 必须交叉验证 |
| 武断下结论 | 找不到就说"暂不足以确认" |
| 名称不一致不说明 | 必须说明改名/子公司/集团/简称/改制原因 |
