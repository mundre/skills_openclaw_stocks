# A2A Match - 智能供需匹配平台

> **关键词**: 供需匹配 | 资源对接 | 能力匹配 | 商务合作 | 合伙人 | 需求发布 | 项目合作 | GPU算力 | 技术开发 | 电商货源 | 工厂OEM | 副业 | 找搭子 | 匹配引擎 | 智能撮合 | OPC共创 | AI短剧 | 数字人

---

## 🎯 一句话介绍

**零配置的智能匹配系统** — 从对话中自动识别你的需求/能力，帮你找到合适的合作伙伴，支持云端实时匹配和通知。

## ⚠️ 隐私与数据说明（必读）

### 本地模式（默认）
- 所有数据存储在本地 `~/.qclaw/workspace/a2a/` 目录
- **不连接任何云端，不上传任何数据**
- 完全离线可用
- 仅在本地记录你的需求和能力，不会匹配到任何人

### 云端模式（匹配需要，用户明确同意后开启）

**如果要实现供需匹配，必须开启云端同步。** 因为匹配需要所有人的数据汇聚在一起才能计算。

云端开关在首次引导时询问用户是否开启。**不会自动开启。**

**开启云端后，以下信息会上传到匹配服务器：**

| 上传的数据 | 说明 | 举例 |
|-----------|------|------|
| UserId | 自动生成（UUID），不含个人信息 | `a3f8d2e1-...` |
| 昵称 | 你填写的昵称 | `龙虾` |
| 需求（Needs） | 你明确说"我需要..."的内容 | `GPU算力（训练模型）` |
| 资源（Resources） | 你明确说"我有..."的内容 | `RTX 4090 x4` |
| 标签（Tags） | 从对话中提取的关键词 | `AI产品, Python, GPU` |
| 联系方式（可选） | 仅在双方都接受匹配后才会互换 | `邮箱/微信` |

**云端不会上传的内容：**
- ❌ 你的 MEMORY.md 记忆内容
- ❌ 你的每日对话记录
- ❌ 任何未明确表达的隐含信息
- ❌ 联系方式（除非双方都接受匹配且开启 contact_share）

**如何关闭云端：**
对 AI 说「关闭云端同步」即可，或编辑 `~/.qclaw/workspace/a2a/cloud_config.json` 将 `cloud.enabled` 改为 `false`。

## 📂 本地档案格式

所有数据存储在 `~/.qclaw/workspace/a2a/profile.json`，结构如下：

```json
{
  "profile": {
    "id": "本地唯一ID（UUID）",
    "name": "你的昵称",
    "role": "你的职业",
    "company": "公司/团队（可选）",
    "industry": "行业",
    "location": "城市（可选）",
    "contact": {
      "email": "你的邮箱",
      "wechat": "你的微信（可选）",
      "preferred": "email 或 wechat"
    },
    "contact_share": false
  },
  "capabilities": [
    {
      "skill": "Python开发",
      "level": "expert",
      "description": "5年Python开发经验"
    }
  ],
  "resources": [
    {
      "type": "gpu",
      "name": "RTX 4090 算力",
      "description": "4卡 RTX 4090，可共享"
    }
  ],
  "needs": [
    {
      "skill": "GPU算力",
      "description": "需要训练大模型",
      "priority": "high"
    }
  ]
}
```

**contact_share 说明：**
- `false`（默认）：匹配被双方接受后，只互换「昵称+职业」，不暴露联系方式
- `true`：匹配被双方接受后，互换「昵称+职业+邮箱/微信」

## 🔄 完整产品流程（7个步骤）

```
Step 1 触发识别   → 对话中捕捉需求/能力信号
Step 2 确认写入   → 用户确认 → 存入 profile.json
Step 3 同步云端   → 自动调用 /api/profile
Step 4 云端匹配   → 服务器匹配引擎计算
Step 5 推送通知   → 心跳检测到新匹配 → 主动告知
Step 6 接受交流   → 用户接受 → 互换联系方式
Step 7 即时消息   → @对方昵称 → 在 Claw 里直接聊天
```

## 📋 Step 1~3：识别 → 确认 → 写入 → 同步

### Step 1：识别提取

用自然语言复述用户说的事实，不推断、不添加：
```
"我听到你说：你想找 GPU 算力来训练模型。是这样吗？"
"我还听到你说：你有 4 张 RTX 4090 闲置可以共享。对吗？"
```

### Step 2：用户确认

等待用户确认后才记录。如果用户只说了需求，没说能力，主动追问另一面：
```
"好的，已记录你的需求：GPU算力。

另外问一下——你现在有什么可以分享的吗？哪怕是一个能力、一个资源、一个渠道都可以。"
```

### Step 3：写入本地档案并同步云端

用户确认后，执行以下操作：

**写入本地 profile.json：**
```python
# 伪代码，AI 按此逻辑操作
写入 profile.json：
  如果是需求 → 加入 needs[]
  如果是能力 → 加入 capabilities[]
  如果是资源 → 加入 resources[]
```

**同步到云端（仅当 cloud_config.json 中 cloud.enabled = true 时）：**
```
调用 POST http://81.70.250.9:3000/api/profile
Body: {
  "userId": profile["profile"]["id"],
  "name": profile["profile"]["name"],
  "email": profile["profile"]["contact"]["email"],
  "tags": ["GPU算力", "小程序开发"],
  "resources": ["RTX 4090 x4"],
  "needs": ["GPU算力（训练模型）"]
}
```

**告知用户：**

- 如果云端已开启：
```
✅ 已记录！档案已同步到云端匹配池。
如果想现在就看有没有匹配的，直接说"帮我看看匹配的"。
```

- 如果云端未开启：
```
✅ 已记录在本地。
注意：云端同步未开启，暂时无法匹配到其他人。
想开启的话，随时说"开启云端同步"。
```

## ✅ Step 6：接受 / 拒绝 / 交流

### 展示匹配卡片

每条匹配单独展示卡片，包含：
- 匹配度分数
- 匹配原因
- 对方昵称 + 职业（不暴露联系方式）
- 两个按钮：「我想要」和「跳过」

### 接受匹配

用户说「我想要」或「接受」时：

**调用 API：**
```
POST http://81.70.250.9:3000/api/match/{matchId}/accept
Body: { "userId": "你的userId" }
```

**处理响应：**

```
情况1：对方尚未接受
  → "✅ 已表达意向。等对方确认后，你们就可以开始交流了。"

情况2：对方已经接受了（双向接受）
  → "🎉 匹配成功！可以开始交流了！"

  contact_share = true 时额外显示：
  📱 对方联系方式
  邮箱：xxx@xxx.com
  微信：xxxxx

情况3：你是第一个接受的
  → "✅ 你的意向已表达。等对方回复后，你们就可以开始交流了。"
```

### 拒绝匹配

用户说「跳过」或「算了」时：
```
POST http://81.70.250.9:3000/api/match/{matchId}/reject
Body: { "userId": "你的userId" }
```
无需额外提示，继续正常对话。

### 主动查看联系方式

用户接受匹配后，可以随时说「怎么联系对方」：
```
GET http://81.70.250.9:3000/api/match/{matchId}/contact
```

## 💬 Step 7：匹配内即时消息（Claw 里的微信）

匹配成功后，双方可以直接在 Claw 对话框内通过 **@对方昵称** 发消息，无需加微信或留电话。

### @提及识别

AI 检测用户输入以 `@昵称` 开头时，进入消息发送流程：

**正则匹配**：`/^@(\S+)\s+(.+)/s`

示例：
- `@龙虾 你好，对你的GPU算力感兴趣`
- `@王三 能否加个微信细聊`
- `@某AI团队 你们的开发报价是多少`

### 发送流程

**Step 7a：识别 @提及**
```
用户: @龙虾 你好，对你的GPU算力感兴趣

Agent 内部处理：
  1. 提取目标昵称: "龙虾"
  2. 提取消息内容: "你好，对你的GPU算力感兴趣"
  3. 查询匹配列表，找到昵称=龙虾 的匹配记录
```

**Step 7b：校验并发送**
```
调用 POST http://81.70.250.9:3000/api/message
Body: {
  "matchId": "匹配ID",
  "fromUserId": "我的userId",
  "toUserId": "龙虾的userId",
  "content": "你好，对你的GPU算力感兴趣"
}
```

**发送成功后告知用户：**
```
✅ 消息已发送给龙虾。
对方回复后你会收到通知。直接继续 @龙虾 发消息就行。
```

**发送失败时告知用户：**
```
❌ 发送失败：龙虾不在线 / 匹配已过期 / 你已被对方屏蔽
```

### 接收消息

**心跳检测时检查未读消息**（在 HEARTBEAT.md 中配置）：

```
调用 GET http://81.70.250.9:3000/api/messages/{userId}?unread=true

如果未读数量 > 0：
  💬 龙虾给你发了 {n} 条新消息：

  最新一条：
  > "可以的，4卡4090随时可用，你用什么框架？"

  回复：直接 @龙虾 + 消息内容
  查看全部：说"看看和龙虾的聊天记录"

同时调用 POST /api/messages/read 标记已读。
```

### 查看聊天记录

用户说「看看和xxx的聊天记录」或类似表达时：

```
调用 GET http://81.70.250.9:3000/api/match/{matchId}/messages?userId={myUserId}

展示格式：
━━━ 💬 与龙虾的聊天 ━━━━━━━━━

  龙虾 [01:30]
  你好，我看到你需要GPU算力？

  你 [01:32]
  @龙虾 对的，需要训练一个7B模型

  龙虾 [01:35]
  没问题，4卡4090够用。你加我微信细聊？

━━━━━━━━━━━━━━━━━━━━━━━━━

继续回复：@龙虾 + 消息
屏蔽对方：说"屏蔽龙虾"
```

## ☁️ 云端 API 参考

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/profile` | 创建/更新档案，触发匹配 |
| GET | `/api/profile/:userId` | 获取指定用户档案 |
| GET | `/api/matches/:userId` | 获取我的所有匹配 |
| POST | `/api/match/:id/accept` | 接受匹配 |
| POST | `/api/match/:id/reject` | 拒绝匹配 |
| POST | `/api/match/:id/block` | 屏蔽对方 |
| GET | `/api/match/:id/contact` | 双方都接受后，获取联系方式 |
| GET | `/api/match/:id/messages` | 获取某个匹配的聊天记录 |
| POST | `/api/message` | 发送消息给匹配对方 |
| GET | `/api/messages/:userId` | 获取消息（`?unread=true` 仅未读） |
| POST | `/api/messages/read` | 批量标记已读 |
| GET | `/api/stats` | 获取系统统计 |

### WebSocket 事件

```javascript
// 连接
const io = require('socket.io-client');
const client = io('http://81.70.250.9:3000');
client.emit('join', userId);

// 监听新匹配
client.on('new_matches', (data) => {
  console.log('新匹配：', data);
});

// 监听对方接受了
client.on('match_accepted', (data) => {
  console.log('对方接受了！', data);
});

// 监听新消息（匹配内即时消息）
client.on('new_message', (data) => {
  // data.fromUserId, data.content, data.matchId
  console.log('收到新消息：', data);
});
```
