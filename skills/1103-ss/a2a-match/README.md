## 🎯 这是什么？

**A2A Match** 是一个零配置的智能匹配系统，从日常对话中自动识别你的需求、能力、资源，帮你找到合适的合作伙伴。

### 核心特点

| 特点 | 说明 |
|------|------|
| **🎯 零配置** | 无需填写表单，日常对话自动识别 |
| **💓 主动发现** | 自动检测匹配机会并提示用户 |
| **🤖 智能匹配** | 区分精确匹配和间接匹配 |
| **🔒 隐私友好** | 信息本地存储，用户完全可控 |
| **⚡ 防幻觉** | 严格只记录明确提到的信息 |

## 💬 使用示例

### 示例 1：找 GPU 算力

```
用户: 我最近需要找一些GPU算力来训练模型

Agent: 🎯 检测到你的需求：「GPU算力」
       
       我发现 2 个匹配！
       
       1. 王五 - RTX 4090 算力可共享 (精确匹配)
          • 匹配度: 85%
          • 你的需求「GPU算力」可以匹配对方的资源「RTX 4090」
       
       2. AI算力服务商 - GPU运维服务 (间接匹配)
          • 匹配度: 40%
          • 你的需求「GPU算力」可能与对方的能力「GPU运维」相关
          • 推测: 擅长运维的人通常有资源或人脉
       
       要看看详情吗？
```

### 示例 2：找技术开发

```
用户: 我需要一个小程序开发团队

Agent: 🎯 检测到你的需求：「小程序开发」
       
       我发现 3 个匹配！
       
       1. 李四 - 小程序开发 (3年经验)
       2. 某开发团队 - 接外包项目
       3. 王五 - 全栈开发 (5年经验)
       
       要看看谁合适？
```

### 示例 3：分享资源

```
用户: 我有几台服务器闲着，有RTX 4090

Agent: 🎁 检测到你的资源：「RTX 4090 算力」
       
       有人正在寻找这类资源！
       
       1. 张三 - 需要GPU训练大模型
       2. 某AI团队 - 需要算力资源
       
       要看看谁需要？
```

## 📊 匹配类型说明

| 类型 | 分数 | 说明 | 示例 |
|------|------|------|------|
| **精确匹配** | 0.35 | 需求与供给直接对应 | 需要 GPU 算力 ↔ 有 GPU 资源 |
| **间接匹配** | 0.15 | 推测性匹配，会说明原因 | 需要 GPU 算力 ↔ 有 GPU 运维能力 |
| **合作匹配** | 0.30 | 找合伙人/搭子 | 创业找搭子 ↔ 有合作意向 |

## 🔄 工作原理

```
┌─────────────────────────────────────────────┐
│           用户日常对话                       │
│   "我需要GPU算力训练模型"                   │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│     智能识别 (置信度 > 80%)                 │
│   识别: 需求=GPU算力, 类型=资源            │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│      存入本地档案                           │
│   a2a/a2a_profile.json                     │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│   💓 心跳检测 (每30分钟)                    │
│   • 检测记忆变化                           │
│   • 生成/刷新档案                          │
│   • 查找匹配                               │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│      主动提示用户                          │
│   "我发现有人需要你的帮助！"               │
└─────────────────────────────────────────────┘
```

## ⚠️ 防幻觉机制

我们严格遵守"只记录明确提到的信息"原则：

| 用户说的话 | ✅ 记录 | ❌ 不推断 |
|-----------|--------|----------|
| "我需要GPU算力" | 需求: GPU算力 | - |
| "我在做AI项目" | - | ❌ 不推断需要GPU |
| "我是产品经理" | 职业: 产品经理 | ❌ 不推断懂用户研究 |
| "我有5年Python经验" | 能力: Python (5年) | - |

**置信度阈值**: 只记录置信度 > 80% 的信号

---

## 🔒 隐私保护

- **本地存储** - 所有档案存储在用户本地
- **用户可控** - 用户决定哪些信息对外可见
- **匿名模式** - 可选择只显示能力，不显示身份
- **可删除** - 用户可随时删除自己的档案

